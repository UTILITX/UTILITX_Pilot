# UTILITX Engineering Governance & Ways of Working
**Version:** 1.0  
**Last Updated:** 2025-11-28  

---

## 1. Source Control & Branching
- GitHub is source of truth  
- Protected `main` branch (no direct commits)  
- All changes via feature branches  
- PR review required  

---

## 2. Pull Request Standards
Every PR **must include**:

- Purpose/description  
- Linked issue  
- Screenshots for UI changes  
- Acceptance criteria  
- Security/privacy considerations  

---

## 3. Release Process
- Weekly deployment to **stage**  
- Monthly deployment to **prod**  
- Hotfixes require reviewer approval  

---

## 4. Coding Practices
- TypeScript-first  
- No inline secrets  
- Environment variables for configuration  
- Strive for modular and documented components  

---

## 5. Logging & Monitoring
Logged events:

- WorkArea created/edited  
- Record uploaded/deleted  
- Sharing links created  
- Permission changes  
- Authentication events  

---

## 6. Access Management
- **Admin:** Founders  
- **Editor:** Core contributors  
- **Viewer:** Advisors/demos  
- Quarterly access review performed
